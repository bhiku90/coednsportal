DNS Privacy


A key function of the DNS is to map human readable names (e.g. www.cdac.in) to IP addresses that requires to connect to a particular web site. As the data in the DNS is public, individual transactions made by an end user should not be public. However, DNS queries are sent in clear text (using UDP or TCP) which means passive eavesdroppers can observe all the DNS lookups performed.


DNS-OVER-TLS

One proposed solution is to change DNS to use TLS to send queries, i.e. DNS_OVER_TLS. TLS is an encrypted protocol which will prevent passive reconnaissance of the network revealing users' DNS queries. This will also allow users to validate the server they choose for their DNS service to make sure that they are using a provider who has a good privacy policy for how they handle the user data.

QNAME minimization

QNAME minimization (query name minimization) is a way to resolve domain names without having to send the full request to every authority in the chain.
For example, if we are searching for www.cdac.in, we are having three authorities, Root server who has .in, TLD has cdac.in, and sTLD has www.cdac.in. Then we have to ask Root server for .in, TLD for cdac.in and sTLD for www.cdac.in, Instead of leaking the full www.cdac.in to every authority.
